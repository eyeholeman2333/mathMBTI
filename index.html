MathITI 维度解释 它不衡量能力高低，而描述你更自然的思考方式
D / C 对象偏好 · Object Preference
世界是「结构的集合」还是「连续的流」？
🧩 D — 离散
  • 图、序列、状态机、token、网络结构
  • 关注"关系"与"结构"，而不是平滑变化
  • 系统 = 明确的状态 + 转移规则
「这个问题可以转成图问题吗？」
🌊 C — 连续
  • 函数、流形、向量场、微分方程
  • 关注"变化"与"演化过程"
  • 系统 = 在空间中流动的整体
「这个系统的演化方程是什么？」
C / E 推理风格 · Reasoning Style
理解是「构造能力」还是「结构洞察」？
🛠 C — 构造
  • 偏好算法、构造方法、可计算过程
  • 理解 = 能实现 / 能生成 / 能复现
  • 不满足"存在"，必须"找到"
「这个解能不能写成算法？」
🧭 E — 存在
  • 偏好证明、结构、极限与边界
  • 理解 = 知道何时成立、为何成立
  • 接受非构造性方法（紧性、对偶性）
「它为什么一定存在？」
P / D 信息态度 · Information Attitude
不确定性是「本质结构」还是「待消除的噪声」？
🎲 P — 概率
  • 用分布、随机过程描述系统
  • 接受"结果是随机的"
  • 更关注整体统计规律
「这其实是一个分布问题」
📏 D — 确定
  • 系统由明确函数或规则决定
  • 不确定性 = 信息不足或噪声
  • 偏好精确解和可重复性
「能不能把随机性去掉？」
G / L 结构视角 · Structural View
理解 = 「整体结构 + 表示切换」还是「局部机制 + 精细分析」？
🌐 G — 全局
  • 关注结构、对称性、谱、拓扑
  • 善于切换表示（时域 ↔ 频域）
  • 先看"大图景"，再看细节
「整体结构是什么？有没有更好的表示？」
🔍 L — 局部
  • 关注点态性质、局部展开、微观行为
  • 逐步分析，逐点理解
  • 强调精确控制与细节严谨性
「在这个点附近发生了什么？」
四个维度的核心问题
D vs C世界是「结构」还是「流」？
C vs E理解是「构造」还是「证明」？
P vs D不确定性是「本质」还是「噪声」？
G vs L从「整体」还是「局部」进入问题？
